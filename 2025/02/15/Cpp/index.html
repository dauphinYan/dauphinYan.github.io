组合

组合（Composition）是一种面向对象编程的设计原则，它描述了一种“拥有”关系（”has-a” relationship）。通过组合，一个类可以包含另一个类的对象作为其成员变量，从而实现代码的重用和功能的扩展。

构造函数执行顺序

  1. 基类构造函数：如果一个类继承自一个或多个基类，那么这些基类的构造函数会首先被调用。如果有多个基类，则根据它们在派生类继承列表中的顺序来调用其构造函数。
  2. 成员变量的构造函数：接下来，在进入派生类（即当前定义的对象所属的类）的构造函数体之前，会先调用属于该类的所有成员变量的构造函数。成员变量的构造顺序依据它们在类中声明的顺序，而不是它们在初始化列表中的顺序。
  3. 派生类构造函数：最后，当所有基类和成员变量的构造函数都执行完毕后，才会执行派生类（即当前类）的构造函数主体部分。

异常转播过程

  1. 抛出异常：当程序执行过程中遇到错误或特殊情况时，可以通过throw关键字抛出一个异常对象。例如：

  2. 栈展开（Stack Unwinding）：一旦抛出异常，程序会开始回溯调用栈，寻找最近的匹配catch块来处理这个异常。这个过程称为栈展开。在栈展开期间，所有位于抛出点和匹配catch块之间的局部对象都会被销毁，即它们的析构函数会被调用。

  3. 捕获异常：当找到合适的catch块后，控制流会转移到该catch块，并执行其中的代码。如果没有任何catch块匹配当前抛出的异常类型，程序将调用std::terminate()并终止执行。

一个异常正在传播中的情况

在一个异常已经处于传播过程中时，意味着程序正在执行栈展开步骤，试图找到适当的catch块来处理这个异常。在此期间，如果另一个异常被抛出（例如，在析构函数中），就会导致问题：

  • 双重异常：如果在栈展开过程中，另一个异常被抛出（如在析构函数中抛出异常），C++标准规定这种情况会导致调用std::terminate()，从而立即终止程序。这是因为C++运行时环境不支持同时处理两个未解决的异常。

如何避免

为了避免上述问题，可以采取以下措施：

  1. 确保析构函数不抛出异常：尽量设计析构函数使其不抛出任何异常。如果必须处理可能抛出异常的操作，可以在这些操作周围使用try-catch块来捕获并处理异常。
  2. 使用智能指针和其他RAII技术：利用C++提供的智能指针（如std::unique_ptr和std::shared_ptr）和其他RAII类来自动管理资源，减少手动管理资源带来的复杂性和潜在错误。

C++11新特性

右值引用

🎯 1. 简明定义

面试官问： “你了解右值引用吗？”

回答：
“是的，右值引用（&&）是 C++11 引入的一个特性，主要用于绑定到右值（即临时对象），以支持移动语义和完美转发。它的核心作用是优化资源管理，避免不必要的深拷贝，从而提高程序的性能。”

🚀 2. 移动语义的作用

面试官可能追问： “右值引用的实际应用是什么？”

回答：
“右值引用的主要用途之一是实现移动语义。当对象资源比较大（比如一个动态数组或文件句柄）时，传统的拷贝构造会有较高的开销。使用右值引用时，可以将资源的所有权从一个对象转移到另一个对象，避免了资源的重新分配和拷贝操作。”

⚡ 4. 面试加分点：实际场景应用

如果面试官深入探讨，可以补充一个实际应用场景：

“在标准库中，std::vector 的 push_back 方法就用到了右值引用。如果传入一个右值对象，容器会使用移动构造而不是拷贝构造，从而提高性能。”

具体解释：

C++11 中，std::vector 为 push_back 提供了两个重载版本：

✅ 传入左值时：

  • 使用 拷贝构造函数（const T&）。
  • 会创建一个对象的副本，复制所有数据。

🚀 传入右值时：

  • 使用 移动构造函数（T&&）。
  • 不会复制数据，而是直接转移资源（例如，指针所有权的转移）。
